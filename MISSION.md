## How agents use this file

`MISSION.md` is the human-owned statement of intent, read at the start of every
session (see `AGENTS.md`). It sets **direction, not authority**: work still
flows only through approved feature packs in `.specify/specs/`, typed task
contracts, exact write scopes, and the verification gates. Agents never edit
this file; only humans do. When the mission changes, update this file, commit,
and re-run the kickoff procedure (`agent_docs/kickoff.md`; `/kickoff` in Claude
Code) to re-derive the roadmap.
